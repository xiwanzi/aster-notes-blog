# 抽卡模拟器实现说明

## 视觉方向

界面采用原创科幻二次元世界观「星汐回响」。美术由本项目单独生成，不复用现有游戏角色、标识或界面素材。大厅以星门观测台为场景，通过暗蓝玻璃 UI、青色能量和淡金高稀有度反馈形成统一的视觉语言。

## 动画状态机

```text
idle → charging → flight → burst → results
```

- `charging`：星门多层圆环旋转、粒子向中心聚合。
- `flight`：镜头进入折光航道，彗星与星点产生高速位移。
- `burst`：根据本批最高稀有度改变场景色，闪光后显示判定文案。
- `results`：单抽使用角色大图揭示；九连使用错峰卡牌翻转，可继续查看单卡详情。

所有运动主要通过 `transform`、`opacity` 和合成层完成；`prefers-reduced-motion` 会把长动画缩短为直接揭示结果。

## 抽取规则

- 五星基础概率 1.6%，四星概率 12%。
- 60 抽五星保底，第 50 抽开始逐次提升五星概率。
- 九连至少出现一项四星或以上奖励。
- 当前会话的首次抽取固定包含五星，用于演示最高规格动画。

本项目是纯前端模拟器，不包含充值、真实货币、账号系统或服务器随机数。

## 性能策略

- 角色 PNG 已转为高质量 WebP，四张主要美术合计约 1.3 MB。
- 首屏只展示背景和当期角色，其余角色在浏览器空闲阶段预加载。
- 桌面九连为横向九卡，窄屏调整为 3×3 网格。
- 音效使用 Web Audio API 动态合成，避免加载音频资源。
